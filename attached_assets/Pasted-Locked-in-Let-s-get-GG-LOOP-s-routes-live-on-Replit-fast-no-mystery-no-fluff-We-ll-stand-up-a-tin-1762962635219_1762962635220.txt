1) Spin up the server (Express)

In Replit, create a Node.js repl (or use your existing one) and add these files.

package.json

{
  "name": "ggloop",
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "start": "node server.js"
  },
  "dependencies": {
    "express": "4.19.2",
    "ejs": "3.1.10"
  }
}


server.js

import express from "express";
import path from "path";
import { fileURLToPath } from "url";

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

const app = express();

// view engine
app.set("view engine", "ejs");
app.set("views", path.join(__dirname, "views"));

// static assets (css/images)
app.use("/static", express.static(path.join(__dirname, "public")));

const pages = [
  { slug: "launch", title: "Launch Checklist" },
  { slug: "twitch-outreach", title: "Twitch Streamer Outreach" },
  { slug: "sponsor-outreach", title: "Sponsor Outreach Kit" },
  { slug: "investor-outreach", title: "Investor Outreach Kit" },
  { slug: "roadmap", title: "6-Week Validation Roadmap" }
];

// home hub
app.get("/", (req, res) => {
  res.render("home", { pages });
});

// generic page handler for /launch, /twitch-outreach, etc.
app.get("/:slug", (req, res, next) => {
  const p = pages.find(x => x.slug === req.params.slug);
  if (!p) return next();
  res.render("page", { page: p });
});

// 404
app.use((req, res) => res.status(404).render("404"));

const PORT = process.env.PORT || 5000;
app.listen(PORT, () => console.log(`GG LOOP live on ${PORT}`));

5) Hook up your custom domain (Namecheap → Replit)

In Replit, click Deployments (or the globe icon) → Add Domain → enter ggloop.io.

Replit gives you DNS records (often a CNAME to something like cname.repl.co and a TXT for verification).

In Namecheap → Advanced DNS, add those exact records. Save.

Back in Replit, click Verify. Propagation can take from 5 min to a couple hours.

Once verified, visiting https://ggloop.io/launch should render your page.

6) Sanity checks (don’t skip)

PORT: Replit sets process.env.PORT. We’re using that—✅

Always-on: If you’re not using Deployments, turn on “Always On” (if available) or use Deploy to keep it live.

HTTPS only: Use your domain’s https:// links in outreach.
